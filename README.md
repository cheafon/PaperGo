# PaperGo
A full-stack agent web client assists researchers in obtaining trends and information related to papers, and more than just that.

## Feature
总而言之：所有的功能都要符合人类的直觉，现在所有的系统经过调研，都是功能太多，且入口难找，没有聚焦，这个系统就是小而精，就从base line做起。
1. chat，对各个问题都能智能解析，从而准确的route到对应的功能。 通过api接口去调用，前台用echart对接即可，用户问题完毕之后，可以选择性的生成图表：柱状图、饼形图
   
    痛点：没有心流的感觉，没有“心路历程”提示，比如我们人类去问一个问题，肯定是先去google，然后点进什么，发现没有，再退回来，然后再点，然后发现了,通过发现发散了一个思维，然后，又去做什么，最终恍然大悟“原来是这样”，要把心路历程带给用户。
   
    难点：上下文管理，错误的处理，prompt的设计，问题覆盖的广度，以及对HuMan In Loop的设计。
2. 综述生成（学习Zone）：
   
   痛点：不全面，不直观，知识网络树，形成一个线性故事，产生一个可播放，可回溯，可交互的动画，非常直观的展示从某一年开始，XX作者做出了贡献，转折点（引用网络），现状是如何，难点如何，能否给出一个新的创新点，利用这个网络去学习新的知识。（不知道这个领域的背景，学习有阻碍）
   
   难点：UI交互的设计，创新点的生成，这要用知识图谱的路径算法等等，这里比较有挑战性
   
3. 生成论文：
   
   痛点：没有心路历程，应该是我们一想到写一个论文应该是什么样的
      这个有一个文献网络的基础功能，这个是我自己设计的小算法，可以通过某一篇文章找到该领域下的所有主流热点paper列表，然后用户可以维护更新这个列表，作为自己的论文生成的context，结合自己的创新点，生成模板。然后使用这个模板去不断地迭代，大模型可以记录这些迭代，这些所有迭代的过程也要参与分析，作为任何下一次生成的方向。
   
4. AnyWhere全局
   
   痛点：目前市面上大部分的web，导航太多，传统产品，太多入口，只能搜索文章，不能搜索页面的元素，有一个全局的agent可以用来进行搜索，搜索历史记录，搜索按钮导航，页面尽量简洁，留给以上三个功能。
